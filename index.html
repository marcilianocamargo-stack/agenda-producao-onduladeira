3. Produção por pedido

0 selecionados
Cada aba do Excel é um pedido — um cliente pode ter mais de um (ex: ABAPEL, ABAPEL3), cada um com sua própria data. Marque as caixinhas de várias OFs (de um ou mais pedidos — ou marque o pedido inteiro no cabeçalho) e toque em "📌 Reservar data" pra agendar todas de uma vez, ou em "Marcar produzidos" pra dar baixa em tudo. Arraste um pedido (⠿) para mudar a ordem entre pedidos. Toque no nome do pedido para abrir e reordenar as OFs dele. "Aguardar" tira da projeção sem apagar; o botão "Produzido" individual serve pra registrar produção parcial (não coube tudo na carreta).
Pedidos que existem mas não devem entrar na projeção de capacidade ainda (ex: aguardando fechar carga para uma região). Não contam nos dias da agenda. Toque em "Liberar p/ fila" quando puder produzir.

📅 Datas reservadas — consulta rápida

Pra responder na hora quando um vendedor ligar: quanto já está reservado em cada data e quanto ainda cabe. "Lotado" significa que essa data já não tem mais espaço livre.

4. Publicar

Como criar o token (1x): no GitHub, vá em Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token. Marque "Only select repositories" → escolha agenda-producao-onduladeira → em Permissions, ative Contents: Read and write. Copie o token e cole acima.

O token fica salvo só neste navegador/dispositivo (nunca é enviado a mais ninguém além do próprio GitHub). Remover token salvo.
Alternativa sem token: baixa o arquivo pra você subir manualmente no GitHub, igual ao fluxo do Romaneio.
